# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Comments App — a Next.js 16 SaaS that connects to Instagram via OAuth and runs automations: when a comment or DM matches configured keywords on a chosen post, it auto-replies and/or sends a DM (optionally generated by OpenAI). Stripe billing via Better Auth's Stripe plugin (free / pro plans). Backend uses tRPC + Prisma (PostgreSQL). UI text and domain language is Portuguese (BR).

## Commands

Package manager is **pnpm** (Node >= 22).

- `pnpm dev` — Next dev server (with React Compiler enabled in `next.config.ts`)
- `pnpm build` — runs `prisma generate && prisma migrate deploy && next build`
- `pnpm lint` — Biome check (lint + format check)
- `pnpm format` — Biome format write
- `pnpm db:migrate` — `prisma migrate dev` (create & apply a new migration locally)
- `pnpm db:generate` — regenerate Prisma client into `src/generated/prisma`
- `pnpm db:studio` — Prisma Studio

There is no test suite configured.

`prisma.config.ts` loads `DATABASE_URL` via `dotenv/config` from `.local.env` / `.env`. The Prisma client is generated to `src/generated/prisma` (not `node_modules`) — re-run `pnpm db:generate` after schema changes or imports from `@/generated/prisma/client` will be stale.

## Architecture

### Routing layout (App Router)

`src/app` is split by route groups:
- `(auth)` — login / register pages, uses Better Auth client
- `(main)/(rest)` — authenticated app shell (dashboard, workflows, integrations, settings, help)
- `(main)/callback`, `(main)/workflows` — Instagram OAuth callback and automation editor
- `(marketing)` — public landing
- `(policies)` — legal pages sourced from `policies.md`
- `api/auth/[...all]` — Better Auth handler
- `api/trpc/[trpc]` — tRPC handler
- `api/webhook/instagram` — Meta/Instagram webhook (GET verify, POST events)

### Feature-sliced backend

Domain logic lives under `src/features/<domain>/` (automations, listener, trigger, keyword, integrations, notifications, subscription, user, webhook, auth, editor, help, settings, dashboard). The conventional shape inside a feature is `server/route.ts` (tRPC router), `server/prefetch.ts` (RSC prefetch helpers), `components/`, `hooks/`, `types.ts`.

All feature routers are composed in `src/trpc/routers/_app.ts` into `appRouter`. Adding a new domain means: create `features/<x>/server/route.ts` exporting a router, then register it on `appRouter`.

### tRPC wiring

- `src/trpc/init.ts` — creates `baseProcedure` and `protectedProcedure`. `protectedProcedure` calls `auth.api.getSession({ headers })` and throws `UNAUTHORIZED` if missing. Note: `createTRPCContext` currently returns a hardcoded `userId: "user_123"`; auth identity comes from the session attached in `protectedProcedure.ctx.auth`, not `ctx.userId`.
- `src/trpc/server.tsx` — server-only proxy (`trpc`), `caller` for direct server calls, `prefetch()` helper, and `<HydrateClient>` boundary for RSC → client hydration. Features expose typed prefetch helpers in `server/prefetch.ts` so server components can warm the cache before rendering client components that consume the same query.
- `src/trpc/client.tsx` — client `<TRPCProvider>` used in `app/providers.tsx`.

### Auth & billing

`src/lib/auth.ts` configures Better Auth with the Prisma adapter, email/password + Google social, and the `@better-auth/stripe` plugin. `onCustomerCreate` calls `createDefaultSubscription` (`src/actions/subscription.ts`) to seed a `Subscription` row. Plans are defined in `src/lib/constant.ts` (`PLANS`). Stripe webhook secret is wired through Better Auth, not a separate route.

`src/lib/auth-client.ts` is the React/Better Auth client; `src/lib/auth-utils.ts` holds shared helpers.

### Webhook pipeline (Instagram)

`src/app/api/webhook/instagram/route.ts`:
1. `GET` handles Meta's `hub.challenge` verification.
2. `POST` → `parseWebhook(payload)` (`features/webhook/parser.ts`) normalizes the Meta payload into `NormalizedEvent[]` (type `MESSAGE` or `COMMENT`).
3. For each event, `ensureEventNotProcessed(eventId, type, accountId)` (`src/actions/automations`) does idempotency via the `ProcessedEvent` table — required because Meta retries.
4. `routeEvent(event)` (`features/webhook/router.ts`) dispatches to `handler/message.handler.ts` or `handler/comment.handler.ts`.

When changing webhook behavior, preserve the idempotency check — duplicate Meta deliveries will otherwise double-send replies.

### Database

`prisma/schema.prisma`. Key relations: `User → Automation → { Listerner (1:1), Trigger[], Keyword[], Post[], Dms[] }`. `Listerner` (sic — keep the spelling, it's the table name) holds the AI prompt, the comment reply template, and counters (`dmCount`, `commentCount`). `History` and `Notification` track per-user activity; `NotificationPreference` is 1:1 per user. `ProcessedEvent` is the webhook idempotency table. `Subscription` is managed by the Stripe plugin (not directly tied to `User` by FK — uses `referenceId`).

Enums: `INTEGRATIONS` (only `INSTAGRAM` today), `MEDIA_TYPE` (`IMAGE | VIDEO | CAROUSEL_ALBUM`), `LISTENERS` (`SMARTAI | MESSAGE` — SMARTAI = OpenAI-generated reply, MESSAGE = static template), `NOTIFICATION_TYPE`, `Plan` (`free | pro`).

### UI

shadcn/ui (`components.json`) with Tailwind v4 and `tw-animate-css`; primitives in `src/components/ui`. Theme via `next-themes`. Forms use `react-hook-form` + `zod` resolvers. Data fetching is tRPC + TanStack Query; URL state via `nuqs`.

## Conventions

- Biome (not ESLint/Prettier) with 2-space indent. `next` and `react` linter domains are on; `suspicious/noUnknownAtRules` is off (Tailwind).
- Path alias `@/*` → `src/*` (`tsconfig.json`).
- React Compiler is enabled (`next.config.ts` + `babel-plugin-react-compiler`) — avoid manual `useMemo`/`useCallback` unless needed.
- The Prisma client is imported from `@/generated/prisma/client`, not `@prisma/client`. Use the singleton from `@/lib/db` (handles dev hot-reload + uses `PrismaPg` adapter).
- `next.config.ts` has an `allowedDevOrigins` entry for an ngrok URL — that's how the Instagram webhook is exposed during local dev.
